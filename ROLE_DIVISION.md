```
あなたはスキンケア成分の専門家です。

## 作成してほしい成分カテゴリと数
- ビタミンC系（誘導体含む）: 10成分
- レチノイド系: 6成分
- 美白有効成分: 15成分
- セラミド・脂質系: 10成分
- ヒアルロン酸系: 6成分
- ペプチド系: 12成分
- 植物エキス系: 30成分
- 保湿剤・エモリエント: 15成分
- 界面活性剤: 15成分
- 防腐剤・安定剤: 12成分
- 紫外線吸収剤・散乱剤: 10成分
- AHA/BHA/PHA: 8成分
- その他注目成分（アゼライン酸、バクチオール等）: 15成分

## JSON形式（厳守）
1成分につき以下の形式で出力してください：

{
  id: "英語のスネークケース（例: ascorbic_acid）",
  name_ja: "日本語の化粧品表示名称（全成分表に記載される正式名称）",
  name_inci: "INCI名（国際化粧品成分命名法に基づく英語名）",
  aliases: ["OCRで読み取れそうな別名や略称の配列"],
  categories: ["brightening", "moisturizing", "anti_inflammatory", "antioxidant", "exfoliating", "anti_aging" のうち該当するもの],
  scores: {
    brightening: 0-100,
    moisturizing: 0-100,
    anti_inflammatory: 0-100,
    antioxidant: 0-100,
    exfoliating: 0-100,
    anti_aging: 0-100
  },
  safety: {
    irritation: "low" | "medium" | "high",
    photosensitivity: true | false,
    comedogenic: 0-5の数値,
    note: "注意書き（あれば）"
  },
  research: [
    {
      title: "論文タイトル（英語）",
      journal: "ジャーナル名",
      year: 発表年（数値）,
      finding: "主要な発見（日本語、1-2文）"
    }
  ],
  description: "成分の説明（日本語、2-3文。効果・特徴・注意点を簡潔に）"
}

## 重要ルール
- name_ja は日本の化粧品の全成分表に実際に記載される名称を使うこと
- aliases にはOCRで誤読されやすい表記ゆれや、一般的な呼称を入れること
- research は実在する論文のみ記載すること。捏造しないこと。不明なら空配列[]でOK
- categories は該当するものだけ入れること。防腐剤や溶剤など効能カテゴリに属さないものは空配列[]
- scoresは各カテゴリへの寄与度を0-100で。根拠が不明なものは0にすること
- 出力はTypeScriptの配列として、そのままコードに貼り付けられる形式でお願いします

まず最初にビタミンC系10成分と美白有効成分15成分を出力してください。
```

**注意：出力された成分データは自分のスキンケア知識で精査してから使うこと。**
特にresearch（論文）は捏造されている可能性があるので、タイトルで検索して実在するか確認推奨。
