# SkinLab 成分データベース作成プロンプト

以下のプロンプトを他のAIに渡して、成分データのExcel作成を依頼してください。

---

## プロンプト本文

あなたは化粧品成分のデータベースをExcelで作成するアシスタントです。
スキンケアアプリ「SkinLab」の成分解析エンジンで使用するデータを作成します。

### ゴール
化粧品に頻出する成分を **200〜500件** Excelファイルにまとめてください。
このExcelは最終的にJSON（TypeScript）に変換してアプリに組み込みます。

---

### Excelのシート構成

#### シート1: `ingredients`（メイン）

| 列名 | 型 | 説明 | 入力例 |
|------|----|------|--------|
| `id` | 文字列 | 一意のID（英数字+アンダースコア、小文字） | `ascorbic_acid` |
| `name_cosmetic` | 文字列 | **化粧品表示名称**（日本化粧品工業連合会の表示名称に準拠） | `アスコルビン酸` |
| `name_quasi_drug` | 文字列 | **医薬部外品名**（医薬部外品原料規格の名称）。該当しない場合は空欄 | `L-アスコルビン酸` |
| `name_inci` | 文字列 | INCI名（国際命名法） | `Ascorbic Acid` |
| `aliases` | 文字列 | 別名・通称をパイプ `\|` 区切り。**OCRマッチング精度に直結するため、実際の製品裏面に書かれうる表記ゆれを網羅すること** | `ビタミンC\|ピュアビタミンC` |
| `ingredient_type` | 文字列 | 成分分類（下記5種から1つ選択） | `有効成分` |
| `categories` | 文字列 | 効能カテゴリをパイプ `\|` 区切り（下記6種から選択、複数可） | `brightening\|antioxidant\|anti_aging` |
| `irritation` | `low` / `medium` / `high` | 刺激レベル | `medium` |
| `photosensitivity` | `TRUE` / `FALSE` | 光感受性（日光で悪化するか） | `TRUE` |
| `comedogenic` | 数値 0-5 | コメドジェニック指数（0=低い, 5=高い） | `0` |
| `safety_note` | 文字列 | 安全性に関する注意事項（日本語）。空欄可 | `高濃度では刺激あり。朝使用時は日焼け止め必須。` |
| `description` | 文字列 | 成分の概要説明（日本語、50-100文字程度） | `最も研究が豊富なビタミンC。直接型で即効性が高いが、安定性が低く酸化しやすい。` |

#### シート2: `research`（研究論文）

| 列名 | 型 | 説明 | 入力例 |
|------|----|------|--------|
| `ingredient_id` | 文字列 | シート1の `id` に対応 | `ascorbic_acid` |
| `title` | 文字列 | 論文タイトル（英語） | `Topical L-Ascorbic Acid: Percutaneous Absorption Studies` |
| `journal` | 文字列 | 掲載誌名 | `Dermatologic Surgery` |
| `year` | 数値 | 発表年 | `2001` |
| `finding` | 文字列 | 主な知見（日本語、1-2文） | `15%濃度で皮膚組織中のVC量が最大化` |

※1成分につき0〜3件の論文を記載。有名な成分（レチノール、ナイアシンアミド等）は2-3件、マイナーな成分は0-1件で可。

---

### 成分分類（`ingredient_type`）— 5種、1つ選択

| 値 | 説明 | 代表例 |
|----|------|--------|
| `水性成分` | 水に溶ける成分。水、保湿剤、水溶性ポリマー等 | 水、グリセリン、BG、DPG、ヒアルロン酸Na、PEG類 |
| `油性成分` | 油に溶ける成分。油脂、ワックス、エステル、シリコーン等 | スクワラン、ホホバオイル、シアバター、ジメチコン、ミリスチン酸イソプロピル |
| `界面活性剤` | 乳化・洗浄・可溶化に使われる成分 | ステアリン酸グリセリル、ラウリル硫酸Na、ポリソルベート60 |
| `有効成分` | **医薬部外品の有効成分として厚生労働省に承認されている成分**。「薬用」製品で「有効成分」欄に記載される成分が該当 | トラネキサム酸、グリチルリチン酸ジカリウム、ナイアシンアミド、L-アスコルビン酸2-グルコシド |
| `その他` | 上記4分類に当てはまらない成分。防腐剤、pH調整剤、着色剤、香料、キレート剤、増粘剤等 | フェノキシエタノール、クエン酸、EDTA-2Na、カルボマー、酸化チタン、香料 |

**注意:**
- 1成分につき **1つだけ** 選択する（最も適切なものを選ぶ）
- 「有効成分」は医薬部外品としての承認があるものに限る。化粧品で効果的な成分でも承認がなければ「有効成分」にしないこと
- 同じ成分でも製品によって有効成分扱いか否か変わる場合がある（例: ナイアシンアミドは医薬部外品の有効成分として承認されているので `有効成分` とする）
- 水溶性の有効成分（トラネキサム酸等）は `水性成分` ではなく `有効成分` を優先する

---

### 効能カテゴリ（`categories`）— 6種、複数選択可

| キー | 日本語 | 代表的な成分例 |
|------|--------|---------------|
| `brightening` | 美白 | ビタミンC系、アルブチン、トラネキサム酸、コウジ酸 |
| `moisturizing` | 保湿 | ヒアルロン酸、セラミド、グリセリン、スクワラン |
| `anti_inflammatory` | 抗炎症 | グリチルリチン酸2K、アラントイン、CICA、パンテノール |
| `antioxidant` | 抗酸化 | トコフェロール、アスタキサンチン、フラーレン、CoQ10 |
| `exfoliating` | 角質ケア | サリチル酸、グリコール酸、乳酸、AHA/BHA |
| `anti_aging` | エイジング | レチノール、ペプチド、バクチオール、幹細胞エキス |

※1成分が複数カテゴリに属するのは普通（例: ナイアシンアミド → brightening, moisturizing, anti_aging）
※ベース成分や防腐剤など、上記効能に該当しない成分は `categories` を空欄にしてよい

### `name_cosmetic` と `name_quasi_drug` の違いについて

日本では化粧品と医薬部外品で成分の表記名称が異なる場合があります。

| | 化粧品表示名称 (`name_cosmetic`) | 医薬部外品名 (`name_quasi_drug`) |
|---|---|---|
| 根拠 | 日本化粧品工業連合会の表示名称 | 医薬部外品原料規格（外原規） |
| 用途 | 「化粧品」の全成分表示に使用 | 「医薬部外品（薬用）」の成分表示に使用 |
| 例1 | アスコルビン酸 | L-アスコルビン酸 |
| 例2 | グリチルリチン酸2K | グリチルリチン酸ジカリウム |
| 例3 | トラネキサム酸 | トラネキサム酸（同じ場合もある） |

**ルール:**
- `name_cosmetic` は **必須**（全成分に記入）
- `name_quasi_drug` は **医薬部外品名がある場合のみ** 記入。ない場合や不明な場合は空欄
- 化粧品表示名称と医薬部外品名が同一の場合でも、両方に記入する
- `aliases` には両方の名称が既に含まれるため、aliasesには **それ以外の** 通称・略称・表記ゆれを入れる

### 収録すべき成分の優先順位

**必須（まず入れるべきもの）:**
1. 日本の化粧品に頻出するベース成分（水、グリセリン、BG、DPG、エタノール、PEG系等）
2. 医薬部外品の有効成分として承認されている主要成分（トラネキサム酸、グリチルリチン酸2K、ナイアシンアミド、各種ビタミンC誘導体等）
3. 主要な美容活性成分（レチノール系、アルブチン等）
4. 保湿成分（ヒアルロン酸各種、セラミド各種、スクワラン、コラーゲン等）
5. 抗炎症成分（アラントイン、CICA/ツボクサエキス等）
6. 酸類（サリチル酸、グリコール酸、乳酸、アゼライン酸等）
7. 防腐剤（フェノキシエタノール、パラベン各種等）
8. 紫外線吸収剤・散乱剤（各種UV filters）
9. 界面活性剤・乳化剤（主要なもの）
10. 油脂・ワックス類（ホホバオイル、シアバター等）
11. 植物エキス系（主要なもの：ハマメリス、カモミール、ローズマリー等）

**あると嬉しい:**
- 韓国コスメに多い成分（プロポリス、ムチン、蜂蜜エキス等）
- 最近流行の成分（バクチオール、CICA、ナイアシンアミド高配合系）
- 増粘剤・安定剤（カルボマー、キサンタンガム等）

---

### aliases（別名）に関する重要な注意

aliasesはアプリのOCR/テキストマッチングの精度に直結します。以下を意識してください：

1. **製品の裏面表記** を想定する（化粧品表記と医薬部外品表記の両方が `name_cosmetic` / `name_quasi_drug` で管理されているため、aliasesにはそれ以外の通称・略称を入れる）
2. **略称・通称** を含める（例: ビタミンC、VB3、VC誘導体 等）
3. **全角・半角のゆれ** はアプリ側で吸収するので不要
4. **表記ゆれ例:**
   - `トコフェロール` と `ビタミンE`
   - `パルミチン酸レチノール` と `レチノールパルミテート`
   - `APPS` と `アプレシエ`

### 出力フォーマット

- **Excel (.xlsx)** で出力
- シート名は `ingredients` と `research` の2つ
- 1行目はヘッダー行
- 文字コードは UTF-8
- パイプ区切り（`|`）のセルは前後にスペースを入れない

### 品質チェックリスト

データ作成後、以下を確認してください：
- [ ] `id` が一意であること
- [ ] `name_cosmetic` が全行に入力されていること
- [ ] `name_quasi_drug` は医薬部外品名がある成分のみ入力されていること
- [ ] `ingredient_type` が5種（`水性成分` / `油性成分` / `界面活性剤` / `有効成分` / `その他`）のいずれかであること
- [ ] `有効成分` に分類した成分が実際に医薬部外品の有効成分として承認されていること
- [ ] `categories` の値が6種のカテゴリキーのいずれかであること
- [ ] `irritation` の値が `low` / `medium` / `high` のいずれかであること
- [ ] `comedogenic` が 0-5 の範囲であること
- [ ] `research` シートの `ingredient_id` がすべて `ingredients` シートの `id` に存在すること
- [ ] 主要成分（ビタミンC系、レチノール系、ナイアシンアミド等）に研究論文が紐づいていること
- [ ] `aliases` にOCRで検出されうる表記ゆれが含まれていること（ただし `name_cosmetic` / `name_quasi_drug` と重複するものは不要）
